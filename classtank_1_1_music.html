Detailed Description

Store and play a music file.

This is intended for playing background music. It reads the file from disk whilst playing it, instead of storing it in memory. This saves memory at the cost of being slower to initially start playing. As such if you want to store and play a sound effect then use tank::SoundEffect.

The sound system supports the following file formats: ogg, wav, flac, aiff, au, raw, paf, svx, nist, voc, ircam, w64, mat4, mat5 pvf, htk, sds, avr, sd2, caf, wve, mpc2k, rf64.

Constructor & Destructor Documentation

tank::Music::Music ( )
default

Construct an empty Music.

Playing an empty music will do nothing and print a warning, so before attempting to play either Music::load() the file or use the other constructor.

tank::Music::Music ( std::string  fileName)

Construct a Music from the file at the given path.

This loads the music file at the given filename. It has the same effect as constructing an empty Music and then calling Music::load() on it.

Supports the following formats: ogg, wav, flac, aiff, au, raw, paf, svx, nist, voc, ircam, w64, mat4, mat5 pvf, htk, sds, avr, sd2, caf, wve, mpc2k, rf64.

tank::Music::Music ( Music const &  )
default

Default copy constructor.

This shallow copies the music file, so is effectively cost-free.
